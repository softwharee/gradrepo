# Graduation project!

Hi and welcome, 

For your graduation we expect to you submit preferably 1 big PDF with all parts of the graduation in there preferably on the same place as this readme (root level of the template)
- Functional design, where you explain your idea and impact you hope to achieve
- Technical design, an hypothetical system overview on how to build and scale the application
- Problem Solving, where you try to find the algorithmically complex part of your application - choose an algorithm - implement a couple and test a few. (actual coding!)

To keep overview it might be a good idea to start multiple documents and only just before submitting combining the documents into one and generating a PDF for it. 
⚠️ just submitting .md or docx files is not allowed. You must submit PDF's! ⚠️

Because for the problem solving you will also need to make some code, you can best create a folder /problem-solving and store your code under there. That is especially handy if you have to implement multiple algorithms and set the side-by-side. 


## planning & feedback
In your graduation prep you've already signed off on regular times to ask feedback and process it. 
Make sure you keep track of the feedback received. If you find it difficult to ask for regular feedback, ask your study coach (SLB'er) to make some time for you. It helps to make an arrangement like sitting down once every 2 weeks. But you are in charge! Don't expect help to automatically come your way.
